4. 集成流程说明

消息存储流程:

  1. 用户发送消息 → 前端调用 storeMessage() 存储用户消息
  2. 调用AI API → 前端调用 /api/chat/stream 获取AI回复
  3. AI回复完成 → 前端调用 storeMessage() 存储AI回复
  4. 更新UI → 前端重新加载消息历史确保数据同步

关键特性:

  • 每个聊天气泡对应数据库中的一条消息记录
  • 用户消息和AI回复分别存储
  • 支持工作流阶段信息存储
  • 自动更新会话时间戳
  • 会话切换时自动加载历史消息
